# The OpenAffiliate Thesis

## Positioning

### Internal (Affitor vision)

> "The open infrastructure for affiliate marketing."

OpenAffiliate is the missing infrastructure layer. Programs scattered across networks, websites, and spreadsheets. No standard format. No API. No way for AI to work with it. We're building the structured, open foundation that the entire affiliate ecosystem runs on.

### Public (for everyone)

> "Find the right affiliate program. Or let AI find it for you."

---

## The Shift

Affiliate marketing used to require months of learning. Research programs. Learn SEO. Write content. Build landing pages. Track conversions. Optimize. Repeat. Most people give up before earning their first dollar.

That's changing.

AI can now do the hard parts. It can research which programs pay well. Write comparison articles. Build landing pages. Send outreach emails. Not someday — today. There are already tools that do each of these steps.

But there's a problem: AI doesn't know which affiliate programs exist, what they pay, or when to recommend them. That information is locked in blog posts, scattered across networks, and buried in marketing pages. There's no single source of truth.

**OpenAffiliate is that source of truth.**

A structured, open-source registry where every affiliate program is described with real data — commission rates, cookie duration, payout terms, restrictions, and clear instructions for when AI should recommend it.

## Who Benefits

**If you have a product** — list your affiliate program for free. Get discovered by partners and AI agents that recommend products to millions of users every day.

**If you promote products** — stop googling "best affiliate programs 2026." Search, compare, and filter programs with real data. Know exactly what you'll earn, how you'll get paid, and what the rules are — before you sign up.

**If you build tools** — use the API, CLI, or MCP connector. Build on top of structured data instead of scraping marketing pages.

**If you use AI** — your AI assistant can already search OpenAffiliate. Ask it to find programs that match your niche. It'll give you real answers, not hallucinated commission rates.

## Where This Is Going

Today, AI helps you *find* the right program.

Soon, AI will help you *do* the work — write the content, build the pages, reach the audience. The tools to do this are being built as open-source skills that anyone can use for free.

The people who pay for expensive marketing software today can use these free tools instead. And the customers who don't want to learn the tools? They'll keep buying SaaS products — which is exactly where affiliate commissions come from.

Everyone wins. Companies get more partners. Partners work faster. Customers find better products. And the whole thing runs on open, structured data that anyone can contribute to.

**That's the future we're building. In the open. From day one.**

## Why Now?

Three things converged:

1. **MCP became a standard.** Model Context Protocol gives AI agents a standard way to connect to external data sources. OpenAffiliate's MCP connector means any Claude, Cursor, or Windsurf user can access the registry with one URL.

2. **Agent skills are real.** 47 open-source affiliate skills already exist (research, content, SEO, landing pages, distribution). The execution layer is ready. What's missing is the structured data layer.

3. **The incumbent gap.** Traditional affiliate networks (Impact, CJ, ShareASale) were built for humans clicking through web dashboards. They have no API strategy for agents. No AGENTS.md. No MCP. The agent economy will route around them.

## Why Open Source?

Because the agent economy runs on trust.

An AI agent recommending products needs to trust its data source. A proprietary, gated registry creates a single point of control — and a single point of bias. Who decides which programs appear first? Who controls what the agent recommends?

Open source means:
- **Transparent data** — every program's YAML file is visible, auditable, version-controlled
- **Community quality** — wrong commission rates get caught and fixed via PR
- **No gatekeeping** — any company can list, any partner can browse, any agent can query
- **Forkable** — if OpenAffiliate ever becomes biased, fork the registry and start fresh

The moat isn't the code. It's the community data, the AGENTS.md coverage, and the network of tools built on top.

## The Bigger Picture

OpenAffiliate is one layer in a three-layer stack:

```
┌─────────────────────────────────────────────┐
│  AGENT SKILLS (Execution Layer)             │
│  47 skills: research, content, SEO,         │
│  landing pages, distribution, analytics     │
│  → Agents DO the affiliate work             │
├─────────────────────────────────────────────┤
│  OPENAFFILIATE (Data & Tools Layer)         │
│  Programs, tools, MCP servers, APIs         │
│  → Agents KNOW what to promote              │
├─────────────────────────────────────────────┤
│  AFFITOR (Platform Layer)                   │
│  Tracking, attribution, commissions,        │
│  payouts, advertiser dashboard              │
│  → The business infrastructure              │
└─────────────────────────────────────────────┘
```

Each layer is independent but composable. You can use OpenAffiliate without Affitor. You can use agent skills without OpenAffiliate. But together, they form the complete stack for the agent-driven affiliate economy.

**The bet:** In 2-3 years, a significant portion of affiliate revenue will be generated by AI agents, not human bloggers. The infrastructure that powers those agents will be as critical as AWS is to web startups today.

OpenAffiliate is building that infrastructure. In the open. From day one.
